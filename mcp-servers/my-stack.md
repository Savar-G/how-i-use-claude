# My MCP Server Stack

The MCP servers I currently have connected to Claude Code, and what each one brings to the table.

## Project Management and Productivity

**ClickUp** -- Full project management access from within Claude. Create and update tasks, track time, manage sprints, and search across the entire workspace without ever opening the ClickUp app.

**Google Calendar** -- Schedule management. Create events, find free time slots, check for conflicts, and respond to invitations. Handy for blocking time around deep work or coordinating meetings while planning sprint work.

**Gmail** -- Search messages, read full threads, and create drafts. Most useful for pulling context from email conversations directly into coding work -- no more tab-switching to find that one thread with the API spec.

**Google Drive** -- Read, search, and create files in Google Drive. Useful for pulling in shared docs, specs, or reference material without leaving the session.

## Design and Frontend

**Figma (2 instances)** -- Two separate Figma connections: one read-only for quick reference, and one with full design context including code connect mappings. Pull design screenshots, variables, component metadata, and layout specs directly into implementation. This is the backbone of the design-to-code workflow.

## Backend and Data

**Supabase** -- Full backend workflow. Run SQL queries, manage migrations, deploy edge functions, manage database branches, and generate TypeScript types. Covers the entire database lifecycle without leaving Claude.

**PostHog** -- Analytics and experimentation. Query event data, manage feature flags, create dashboards, and run experiments. Useful for checking how a feature is performing or setting up tracking as part of implementation.

## Knowledge and Documents

**Obsidian** -- Read, write, and search notes in an Obsidian vault. Acts as a second brain for capturing architectural decisions, project context, ideas, and anything worth remembering across sessions.

**PDF Viewer** -- Read and manipulate PDF files directly. Useful for reviewing contracts, specs, research papers, or any documentation that lives in PDF format.

**Gamma** -- Presentation creation. Generate slide decks from scratch and read existing presentations. Good for turning project updates or proposals into shareable formats quickly.

## Automation and System Control

**Computer Use** -- Desktop automation. Take screenshots, click, type, scroll -- full control over native Mac apps. The fallback for when no dedicated MCP server exists for a tool you need to interact with.

**Chrome Control** -- Browser automation. Navigate pages, execute JavaScript, read page content, and manage tabs. Fills the gap for web-based tools that do not have their own MCP server.

**Claude Preview** -- Preview web apps in a browser environment, take screenshots, and interact with elements. Primarily used for QA testing during frontend development.

**Scheduled Tasks** -- Run Claude agents on a cron-style schedule. Set up recurring automations like daily reports, periodic checks, or scheduled maintenance tasks.

## Discovery

**MCP Registry** -- Search for and discover new MCP servers to install. The starting point when you need to connect a new tool to your setup.

---

This stack is always evolving. The MCP ecosystem is growing fast -- new servers show up regularly, existing ones get better, and the ways they compose together keep expanding. What matters is not the specific list, but the principle: every tool you connect removes friction and keeps you in flow.
